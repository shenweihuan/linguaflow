## 记录

### [#001] 2026-06-08 — 生态位打通：与 Skill 体系的关系厘清
- **用户意图**: 评审发现"LinguaFlow 和现有 Skill 体系（LS/Codex/AIGC）的关系没打通"，用户不知道它在这个生态中的位置
- **调用**: 系统架构分析 + 生态位映射
- **输入**: "README 要新增「定位与愿景」章节，明确 LinguaFlow 不是又一个 skill，而是底层认知科学引擎"
- **输出**:
  - README 新增「定位与愿景」章节：
    - 核心定位："LinguaFlow = 把学习科学的认知原理做成可运行产品"
    - 与三个 Skill 体系的关系图：
      - **Learning System**：应用层——基于 LinguaFlow 的原理构建学习工作流
      - **Codex Learning Skills**：执行层——把 LinguaFlow 的方法变成即时可用的教练技能
      - **AIGC Creative Skills**：创造层——用 LinguaFlow 的理解建构理论指导 AI 创作
    - 技术栈分层：认知科学理论 → 方法论提取 → 可运行 skill → 用户场景
  - 新增「为什么需要 LinguaFlow」章节：3 个痛点 × LinguaFlow 的解决方案
- **效果**: ✅ 完全解决
- **耗时**: 约 28 min
- **改进点**: 关系图是纯文字描述，应该用 Mermaid 或 ASCII 图可视化，降低理解成本

### [#002] 2026-06-08 — 产品化首屏：从学术到落地
- **用户意图**: 明确定位后，需要让 README 从"学术项目介绍"变成"可感知的产品"
- **调用**: 用户体验设计 + 信息架构优化
- **输入**: "首屏新增定位标语，让非学术用户也能在 10 秒内理解价值"
- **输出**:
  - 首屏核心标语：
    > **"把学习科学的认知原理做成可运行产品"**
    >
    > 不是教你认知心理学理论，而是把 Make It Stick、How to Learn、元认知这些书里的精华，
    > 变成你每天都能用的工具和流程。
  - 三段式价值主张：
    1. **对学习者**：不再盲目试错，用经过验证的方法提升效率
    2. **对教育者**：基于科学证据设计教学干预，而不是凭经验
    3. **对 AI 应用开发者**：把认知科学嵌入 AI 系统，让 AI 更懂人类学习规律
  - 「30 秒体验路径**"：3 个入口按钮
    - "我想学得更快" → 对应 LS + Codex
    - "我想理解学习科学" → 对应 LinguaFlow 核心文档
    - "我想构建 AI 教育应用" → 对应 AIGC + LinguaFlow API
- **效果**: ✅ 完全解决
- **耗时**: 约 18 min
- **改进点**: 缺少"理论到实践"的桥梁——用户看完标语后可能还是不知道具体怎么用，建议在首屏加入一个微型案例（如："小明用间隔重复法 3 天背完 100 个单词"）

## 模式分析

**什么情况下用得最好：**
- **生态位厘清任务**（#001）：当系统处于技术栈的中间层时，通过分层关系图能有效解释"它是什么"和"它不是什么"
- **跨学科产品化**（#002）：将学术理论转化为大众可理解的产品语言时，三段式价值主张（对不同用户群体的价值）很有效

**什么情况下容易失败：**
- 暂无明确失败模式（当前样本量仅 2 条）
- 潜在风险：作为"底层引擎"的定位可能导致用户觉得"太抽象而不直接有用"，需要有明确的"即插即用"案例

**下一步优化方向：**
1. **可视化关系图**：用 Mermaid 语法绘制 LS/Codex/AIGC/LinguaFlow 的依赖关系图，嵌入 README
2. **微型案例库**：收集 5-10 个"用户用 LinguaFlow 原理解决了具体问题"的真实故事（如：用费曼技巧准备面试、用间隔重复背单词）
3. **API 文档雏形**：如果未来要支持其他开发者集成，需要定义清晰的接口规范（目前只有概念描述）
4. **理论验证实验**：设计简单的 A/B 测试（如：用/不用 LinguaFlow 方法的记忆效果对比），用数据证明有效性
5. **与其他认知科学项目的对比**：如 Anki（间隔重复）、SuperMemo、Feynman Technique 等工具的关系和差异化
